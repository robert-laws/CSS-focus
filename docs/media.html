Media Queries and Different Screens

Mobile-First Philosophy

The concept of mobile-first web development has to do with beginning the development process with the mobile screen in mind and then moving onto development for larger screens. The reasoning behind this was that developer had been developing for larger screens for which loading network-demanding resources like images and large stylesheets would consume a lot of data, but then be hidden from the mobile display. Using media queries will prevent high-cost asset loading for mobile users.

In Practice

Media Queries for Screen Width

Using media queries is a technique to selectively display content to a user depending on a certain number of criteria that can be defined such as screen minimum and maximum width and screen resolution.

Media queries are defined using a syntax that includes some logic used to determine how to apply the style rules. The basic structure is the following:

CSS
@media (feature: value) {
  rules
}

Not included in the example above is a logical statement for type of media that should be included in the media query. This could be a value like all or screen or print.

For example, if a developer wanted to add styling for devices like a mobile phone, they could create a media query that would style elements specifically for that size of screen.

CSS
@media screen and (max-width: 480px) {
  h1 { font-size: 1.2em; }
}

In this css code, the media query targets screens that have a screen up to the max-width of 480px. If my screen becomes wider than 480px then the rule is no longer applied to the element. Likewise, it's possible to add another feature to the media query.

CSS
@media screen (min-width: 320px) and (max-width: 480px) {
  h1 { font-size: 1.2em; }
}

In this case, the media query targets screens that have a screen width than 320px but only up to a width of 480px. The rule will be applied only to screens that are between 320px and 480px wide.

Media Queries for Resolution

Many devices made within the last few years have much higher quality screens that were available before. They're able to increase the pixel density and therefore display higher resolution images better than older screens. The impact this has is that images that are not optimized for the higher resolution screens may not look very good on newer screens. The difficulty in developing media queries for high resolution screens is the different implmentations across different browsers. use 

CSS
@media only screen and (min-resolution: 200dpi) {
  .logo { background: url(logo@2x.png) no-repeat 0 0; }
}
