Pseudo-Classes and Pseudo-Elements

Foundation Concepts - CSS2

Pseudo-classes provide access to an element's state. Illustrated best with links and the:hover :visited :activestates.

Pseudo-elements, on the other hand, provide access to an elements sub-parts, which can include portions of text nodes, for example with::first-line ::first-letter.

The major advantage of pseudo-classes and pseudo-elements is that they provide ways to hook-up styles without having to add classes without clear symantic value to the markup. Also, if changes are made such as adding more elements to the HTML, additional class names don't need to be added in order for the styling to take effect as desired. The first type considered is CSS3 Structural Pseudo-Classes after that is more.

:First-Child - Pseudo-class (CSS2)

CSS
div p:first-child {
  color: red;
}

:nth-*(n) Pseudo-class (CSS3)

CSS
div p:nth-child(2n + 1) {
  font-style: italic
}

:nth-child(even)

CSS
div p:nth-child(even) {
  font-size: 1.5em
}

:nth-of-type(n)

CSS
div p:nth-of-type(3n) {
  background-color: yellow;
}

:nth-of-last-child(n)

CSS
div p:nth-last-child(-n + 2) {
  color: green;
}

:first-child

CSS
div:first-child {
  text-decoration: underline;
}

:last-child

CSS
div:last-child {
  padding-left: 20px;
}

:first-of-type

CSS
div p:first-of-type {
  border: 1px solid #AAA;
}

:last-of-type

CSS
div p:last-of-type {
  border: 1px solid #F00;
}

:only-child AND :only-type

CSS
div span:only-of-type {
  color: blue;
}

:target

CSS
.example:target {

CSS
p:empty {

:not

CSS
